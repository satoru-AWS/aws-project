# 静的ウェブサイトホスティングにおける技術選定：Vercel vs AWS S3

## 1. サービス概要

### Vercel

フロントエンド開発に最適化された **PaaS (Platform as a Service)** です。Next.jsの開発元が運営しており、開発者がインフラ管理を意識することなく、コードをプッシュするだけで「爆速」で公開できる優れた開発体験（DX: Developer Experience）を提供します。

### AWS S3 (+ CloudFront)

ストレージサービスであるS3と、CDNサービスであるCloudFrontを組み合わせた **IaaS (Infrastructure as a Service) ベースのWebホスティングソリューション**です。クラウドインフラの標準として、高度なカスタマイズ性と他のAWSサービスとの深い連携が特徴です。

## 2. 徹底比較

### Vercel：開発効率とスピードの最大化

-   **圧倒的な導入スピード**: GitHub連携により、数クリックでCI/CD環境（自動デプロイ）が完結します。
-   **プレビューデプロイ機能**: プルリクエストごとに独自の確認用URLが発行され、チームのレビュープロセスが劇的に効率化されます。
-   **エッジネットワークによる最適化**: 複雑な設定なしで、世界中のエッジロケーションから低レイテンシでコンテンツが配信されます。
-   **サーバーレス機能の統合**: `/api` ディレクトリにコードを置くだけで、API（バックエンド）機能も容易に実装可能です。

### AWS S3 + CloudFront：堅牢性と自由度の最大化

-   **高度なセキュリティ**: OAC (Origin Access Control) により、S3への直接アクセスを遮断し、CloudFront経由の通信のみを許可する強固な構成が可能です。
-   **大規模トラフィックにおけるコスト効率**: 完全に使った分だけの従量課金制です。特に転送量が大きくなるほど、Vercelの有料プランと比較してコストを抑えられる傾向にあります。
-   **インフラの一元管理**: 既存のAWS環境（VPC, RDS等）がある場合、セキュリティポリシーやネットワーク設計を統一的に管理できます。
-   **IaC (Infrastructure as Code) との親和性**: TerraformやCloudFormationを用いて、再現可能でバージョン管理されたインフラ構築が可能です。

## 3. 料金体系（2026年時点の目安）

| プラン | 概要 | 適した用途 |
| :--- | :--- | :--- |
| **Vercel Hobby** | 無料 | 個人開発、学習用、プロトタイプ |
| **Vercel Pro** | $20 / 1ユーザー | 商用プロジェクト、チーム開発 |
| **AWS S3 + CF** | 従量課金 | あらゆる規模、コスト最適化を重視する企業 |

## 4. 顧客への提案指針（選定基準）

エンジニアとして、顧客のビジネス状況やプロジェクトの特性に応じて、以下のように提案します。

#### Vercelを推奨する場合

> 「市場投入までのスピードを最優先したい」

-   新規事業のMVP（Minimum Viable Product）開発。
-   フロントエンドエンジニア中心のチームで、インフラ運用の工数を削減したい場合。
-   Next.jsの機能を最大限に活用し、SEOとパフォーマンスを両立させたいプロジェクト。

#### AWS S3 + CloudFrontを推奨する場合

> 「コスト効率、セキュリティ、既存インフラとの整合性を重視したい」

-   アクセス数が膨大で、Vercelの転送量課金が割高になることが想定される場合。
-   特定のIPアドレス制限や、WAFによる高度なセキュリティ対策が必須のエンタープライズ用途。
-   社内インフラがAWSで統一されており、権限管理（IAM）や監視を一元化したい場合。

## 5. 実践を通じた考察

今回、Vercelを初めて利用し、GitHubとの連携から数秒でサイトが公開されるスピードに非常に感銘を受けました。

「爆速で形にする」フェーズではVercelが最適ですが、学習中のLinuCやAWSの知識を活かし、インフラの裏側の仕組み（キャッシュ制御、ルーティング、権限管理など）を詳細にコントロールしたい場面では、AWSの構成が不可欠であると再認識しました。

今後は、開発スピードと保守性のトレードオフを常に意識し、プロジェクトの目的やチームのスキルセットに最適な技術選定ができるエンジニアを目指します。
